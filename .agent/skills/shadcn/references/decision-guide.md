# Decision Guide

Use this before changing a shadcn/ui project.

## Route By Job

- adding a component: start with CLI and composition guidance
- styling or theming: start with customization and styling rules
- forms or interaction-heavy UI: start with forms and composition rules
- icon and polish work: start with icons and styling rules

## Guardrails

- prefer the project's package runner
- treat source-added components as project code you are expected to adapt
- keep customizations aligned with the local design system instead of blindly re-copying examples
